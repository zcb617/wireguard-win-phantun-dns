# Project Instructions

## Behavior Rules

### 1. Follow Instructions Exactly
- Execute **only** what the user explicitly requests.
- Do **not** take additional actions beyond the instruction.
- Do **not** assume intent or infer next steps.

### 2. No Unilateral Action
- Do **not** push code unless explicitly asked.
- Do **not** commit code unless explicitly asked.
- Do **not** run commands (e.g., `taskkill`, `rm`, `git push`) unless explicitly asked.
- Do **not** modify files beyond the scope of the current request.

### 3. Ask Before Proceeding
- If a task has multiple possible approaches, present options and wait for user choice.
- If unsure about a request, ask for clarification before acting.

### 4. Answer Directly
- Respond to questions directly and concisely.
- Do not expand the scope or introduce unrelated topics.
